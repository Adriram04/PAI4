Gestión en DevSecOps
Las herramientas que consideran se podrían ser:
• Static Application Security Testing (SAST)
• Dynamic Application Security Testing (DAST)
• Interactive Application Security Testing (IAST)
• Software Composition Analysis (SCA)
• Security in Infrastructure as Code (IaC)
Estas herramientas se podrían integrar para su gestión en un único sistema, como podría
ser DefectDojo, para facilitar toda la gestión de las vulnerabilidades y encontrar los bugs
de seguridad en etapas más tempranas

Objetivos del PAI4
Pipeline DevSecOps debe cumplir los siguientes objetivos:
1. Definir una Pipeline CI/CD en un proyecto de prueba en un repositorio de código
(SCM)
2. Seleccionar al menos tres herramientas de Testeo de Seguridad (SCA, SAST,
IAST, DAST, Security IaC) para incluirlas en el Pipeline para dotarlos de
capacidades DevSecOps
3. Integrar las herramientas de testeo en el ciclo de vida del desarrollo para hacer
pipeline DevSecOps
4. Desarrollar los tests correspondientes que permitan detectar vulnerabilidades en
cada etapa del Pipeline DevSecOp.
5. Seleccionar una herramienta de gestión de vulnerabilidades para integrar,
priorizar y clasificar las vulnerabilidades de Seguridad que se vayan
encontrando por las herramientas de testeo en el Pipeline DevSecOps. Como
recomendación podría usarse DefectDojo o cualquier otra herramienta que se
considere oportuna.
